# ทรัพย์วราค้าข้าว - ระบบบริหารจัดการค้าส่งข้าวสารและอาหารแห้ง (Wholesale ERP, CRM & POS)

ระบบบริหารจัดการร้านค้าส่งข้าวสารและอาหารแห้ง **"ทรัพย์วราค้าข้าว"** พัฒนาขึ้นด้วยเทคโนโลยีระดับโปรดักชัน เพื่อตอบสนองความต้องการของธุรกิจค้าส่งและแบ่งขายย่อย รองรับการขายเป็นกระสอบและถอดสมการกิโลกรัมโดยอัตโนมัติ ติดตั้งในฐานะ Progressive Web App (PWA) และทำงานร่วมกับ Supabase/PostgreSQL ได้สมบูรณ์แบบ

---

## 🌟 ฟีเจอร์หลัก (Core Features)

1. **หน้าหลัก Dashboard อัจฉริยะ**: 
   - สรุปยอดขายวันนี้ รายรับ รายจ่าย และกำไรสุทธิแบบ Real-time
   - แสดงสถิติกระสอบข้าวคงเหลือในคลัง แยกตามน้ำหนักและข้าวสารแบ่งขายย่อยแบบเปิดปากกระสอบ
   - กราฟแนวโน้ม Recharts แสดงสถิติเปรียบเทียบยอดขาย-รายจ่ายรายวัน รายรับรายเดือน และจัดอันดับสินค้าขายดีที่สุด

2. **ระบบแบ่งขายย่อยข้าวสาร (Automatic Sack-to-Kg Conversion)**:
   - กำหนดค่าแปลงน้ำหนักต่อกระสอบได้อย่างยืดหยุ่นต่อสินค้า (เช่น ข้าวหอมมะลิ 1 กระสอบ = 45 กก. / ข้าวเหนียว 1 กระสอบ = 50 กก.)
   - **ระบบคำนวณตัดสต็อกอัจฉริยะ**: เมื่อทำการขายเป็นกิโลกรัมผ่านตู้ POS ระบบจะตัดยอดกระสอบโดยอัตโนมัติ (ตัวอย่าง: มีข้าวสาร 1 กระสอบ [45 กก.] หากขาย 5 กก. สต็อกจะลดลงเหลือ 0 กระสอบ และเหลือกองแบ่งขาย 40 กก. ทันที)

3. **ระบบขายสินค้าหน้าเครื่อง (POS Terminal)**:
   - ค้นหารายการสินค้ารวดเร็วและฟิลเตอร์แยกตามหมวดหมู่
   - เลือกลูกค้าเข้าระบบเพื่อสะสมยอดขายและจัดส่ง
   - กรอกส่วนลด, เลือกช่องทางชำระเงิน (เงินสด, QR, โอนเงิน, บัตรเครดิต) และออกใบเสร็จรับเงิน (Invoice) ที่รองรับการพิมพ์ออกเครื่องพิมพ์กระดาษความร้อนได้ทันที

4. **ระบบรับสินค้าเข้าคลัง (Stock Receiving / Purchase Orders)**:
   - บันทึกเลขที่บิล วันที่รับของ บริษัทคู่ค้า และชื่อผู้ส่งมอบ
   - อัปเดตยอดคงคลังเพิ่มสต็อกทันทีเมื่อบันทึกใบรับของ พร้อมกับเพิ่มรายการเป็นรายจ่ายโดยอัตโนมัติในบัญชีแยกประเภท

5. **ระบบบริหารจัดการลูกค้า (CRM)**:
   - เก็บรายชื่อ เบอร์โทรศัพท์ และที่อยู่จัดส่งสินค้าโดยละเอียด
   - บันทึกประวัติและสะสมยอดซื้อของลูกค้าแต่ละราย พร้อมติดตรายศสถานะความภักดี เช่น **"ลูกค้าซื้อซ้ำ"** หรือ **"VVIP"**

6. **ระบบจัดการรายรับ-รายจ่าย (Accounting Ledger)**:
   - บันทึกรายการแยกตามหมวดหมู่ประเภทที่หลากหลาย (ซื้อสินค้า, ค่าเช่า, ค่าไฟ, ค่าน้ำ, ค่าแรง, ค่าน้ำมัน, ค่าขนส่ง, ค่าใช้จ่ายอื่น)
   - สรุปผลประกอบการรวมของร้านค้า และคิดคำนวณกำไรสุทธิสะสม

7. **ระบบรายงานและการส่งออกข้อมูล (Reports & Exporting)**:
   - คัดกรองข้อมูลสรุปรายวัน รายเดือน และรายปี
   - ปุ่มส่งออกรายงานด่วนไปยัง **CSV**, **Microsoft Excel** (พร้อมรหัส UTF-8 BOM ที่อ่านภาษาไทยได้สมบูรณ์โดยตัวอักษรไม่เพี้ยน) และพรีวิวจัดเลย์เอาต์พิมพ์ **PDF**

8. **การกำหนดสิทธิ์ผู้ใช้งาน (Role-Based Authorization)**:
   - **Admin (ผู้จัดการ)**: จัดการคลังสินค้า, เพิ่ม/ลบรายชื่อ, บันทึกบัญชี และดูสถิติทางการเงินได้ครบทุกมิติ
   - **Employee (พนักงานขาย)**: ทำรายการขาย POS, ดูยอดสต็อก และประวัติลูกค้าได้ แต่**ไม่สามารถลบข้อมูลธุรกรรมหรือสินค้าใดๆ ออกจากฐานข้อมูลได้**เพื่อความปลอดภัย

## 🚀 ขั้นตอนการติดตั้งใช้งานและเชื่อมต่อกับ Supabase

โปรเจกต์นี้รองรับรูปแบบ **Dual-Mode** โดยจะเริ่มทำงานด้วยฐานข้อมูลประสิทธิภาพสูงบน **LocalStorage** เป็นค่าเริ่มต้นเพื่อให้สามารถกดเล่นใน Live Preview ได้อย่างรวดเร็ว และจะเปลี่ยนเป็นใช้ **Supabase (PostgreSQL ในคลาวด์)** ทันทีเมื่อผู้ใช้ป้อนคีย์การเชื่อมต่อ

### ขั้นตอนการเชื่อมคลาวด์:

1.  **ติดตั้งฐานข้อมูล PostgreSQL**:
    *   ไปที่บัญชี [Supabase Console](https://supabase.com/) และสร้างโปรเจกต์ใหม่
    *   เปิดเมนู **SQL Editor** แล้วสร้าง Query ใหม่
    *   เปิดไฟล์ `/supabase-schema.sql` ในโปรเจกต์นี้ คัดลอกคำสั่งทั้งหมดไปวางใน Supabase SQL Editor แล้วกด **Run**
    *   ฐานข้อมูลจะสร้างตาราง RLS Policies และ Seed ข้อมูลร้านค้าเริ่มต้นให้ทันที

2.  **ระบุตัวแปรสภาพแวดล้อม (Environment Variables)**:
    *   สร้างหรืออัปเดตไฟล์ `.env` ที่โฟลเดอร์รากของโครงการ:
        ```env
        VITE_SUPABASE_URL="https://your-project.supabase.co"
        VITE_SUPABASE_ANON_KEY="your-anon-key-here"
        ```

3.  **รันโปรเจกต์และทดสอบระบบ**:
    ```bash
    npm run dev
    ```

```text
├── src/

│   │   ├── Auth.tsx        # ส่วนเข้าสู่ระบบระบบรักษาความปลอดภัยและการจดจำเซสชัน

│   │   ├── Dashboard.tsx   # รายงานผลประกอบการ สถิติคลัง และกราฟวงจร
│   │   ├── POS.tsx         # ตู้คิดเงินบันทึกการทำธุรกรรมขายและออกใบกำกับภาษี

│   │   ├── StockIn.tsx     # รายการ PO รับของเข้าคลังพร้อมเชื่อมสต็อกสินค้า
│   │   ├── Customers.tsx   # บัญชีลูกค้าสัมพันธ์ CRM สะสมยอดซื้อและตราสถานะความภักดี
│   │   ├── Accounting.tsx  # บันทึกรับ-จ่าย และคิดต้นทุนผลกำไร
│   │   └── Reports.tsx     # จัดการคัดกรองรายงานรายวัน รายเดือน รายปี และส่งออกไฟล์

│   │   └── reportExport.ts # ตัวช่วยแปลงข้อมูลเป็น Excel, CSV และจัด PDF
│   ├── types.ts            # การประกาศโครงสร้างข้อมูล TypeScript

├── package.json            # บันทึก Dependencies ทั้งหมดของโครงการ
└── vite.config.ts          # ไฟล์การตั้งค่าของ Vite
```
